# 04.21 Notion Format Checklist

This checklist captures the visible Notion pattern from the `26.04.21` sample and maps it to what the Markdown publisher can currently produce.

## Implemented blocks

- Page title: date only, e.g. `26.04.22`.
- First paragraph: `최종 수정 일시: YY.MM.DD 06:00 (KST)`.
- Top-level sections:
  - `🗞️ 주요 뉴스 요약`
  - `📚 추천 스토리라인`
  - `🤖 자료 수집`
- Storyline structure:
  - heading 2 for each numbered storyline.
  - quote block for the one-line angle.
  - heading 3 `선정 이유`.
  - heading 3 `슬라이드 구성`.
  - bulleted slide flow.
  - material references are written as prose, e.g. `참고 자료: A -> B`, not as a separate `자료 배치` bullet.
  - referenced material names should exactly match the headings in `2. 오늘의 이모저모`.
- Source material structure:
  - heading 3 for each material.
  - quote block for the editorial use case.
  - one metadata paragraph such as `출처: Isabelnet · 작성 시점: YY.MM.DD` or `데이터: 시장 데이터 · 수집 시점: YY.MM.DD HH:MM`.
  - source links should display compact names such as `KobeissiLetter`, `Isabelnet`, `Finviz`, `TradingView`, not full URLs.
  - image block when available.
  - direct 1-depth detail bullets. Avoid wrapper bullets such as `볼 포인트` / `주요 내용` unless a section genuinely needs grouping.
- Fixed market section:
  - heading 3 per visual.
  - one metadata paragraph per visual, followed by the image.
  - market observations should use `수집 시점`, because the value depends on when the pipeline observed it.
  - Datawrapper chart titles should include latest values, e.g. `미국 10년물 국채금리: 4.368%`.
  - Datawrapper chart subtitles should use only the observation timestamp, e.g. `YY.MM.DD HH:MM 기준`.
  - Datawrapper chart footnotes should stay empty.
  - Notion metadata should show the actual data source link, e.g. `출처: Yahoo Finance · 수집 시점: YY.MM.DD HH:MM`.
  - implementation notes such as capture status, parsing notes, and pipeline caveats are omitted.
- Feature-stock section:
  - heading 3 includes ticker in parentheses, e.g. `마벨 테크놀로지 (MRVL)`.
  - higher-level materials such as earnings calendar or theme maps remain heading 3.
  - higher-level materials do not need quote blocks unless they have a clear editorial angle.
  - individual ticker cards use markdown heading 4, which the publisher renders as a smaller bold paragraph because Notion API has only heading levels 1-3.
  - individual ticker quote blocks should summarize the actual recent issue/news, not merely why the ticker is worth watching.
  - Finviz issue summary is shown as a direct 1-depth bullet when available.
  - wrapper labels such as `Finviz 출발점` and `Finviz 최근 뉴스` are omitted.
- Bottom pipeline memo:
  - removed from Notion-facing reconstruction pages.

## Publisher limits to remember

- Markdown `>` maps to Notion quote blocks.
- Markdown images are uploaded as Notion image blocks.
- Markdown links are supported. Current reconstruction pages use compact source labels as link text so the visible metadata stays short.
- Basic Markdown tables are converted to native Notion table blocks with a header row.
- Inline code maps to Notion code annotation, but this format avoids code styling unless the content itself needs it.
- Bold text is not yet parsed by `publish_recon_to_notion.py`; avoid relying on `**bold**` for format matching.
- Nested bullets are supported through tab indentation.
- Metadata bullets generated by older recon seeds can be flattened into paragraphs by `build_recon_0421_format.py`; legacy `캡처` becomes `수집 시점`, and legacy `작성 일자` becomes `작성 시점`.

## Current gaps

- The publisher does not create Notion file blocks, callouts, columns, toggles, or rich bold annotations.
- Table row text is not expanded by the local inspector yet, but the block type publishes as native `table`/`table_row`.
- Some 0422 materials still use collected 2026-04-28 sources because this is a pipeline format trial, not a fully date-faithful historical reconstruction yet.
- Feature-stock items now include Finviz daily chart captures and one Finviz issue/news summary where available; a later pass should still select the most relevant item instead of showing every ticker mechanically.
