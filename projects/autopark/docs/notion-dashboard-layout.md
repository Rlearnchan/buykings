# Autopark Notion Dashboard Layout

이 문서는 현재 Autopark가 날짜별 Notion 대시보드를 어떤 고정 형식으로 만드는지 정리한다.

## Page Header

페이지 제목은 `26.04.30`처럼 `YY.MM.DD`를 쓴다.

상단에는 세 줄을 고정으로 둔다.

- `최종 수정 일시`: Notion 발행용 Markdown을 만든 시각
- `뉴스/X 수집 구간`: 뉴스, X, 주요 캡처 수집이 실제로 돌았던 구간
- `시장 데이터 기준`: 시장지표 Datawrapper 차트의 기준 시각

시각 값은 Notion에서 눈에 띄도록 코드 텍스트로 표기한다.

## 1. 주요 뉴스 요약

상단 요약은 3줄을 원칙으로 한다.

- 오늘의 메인 thesis
- 보조 관찰 1
- 보조 관찰 2

여기는 자료를 길게 설명하는 곳이 아니라, 진행자가 오늘 시장을 어떤 틀로 볼지 정하는 자리다.

## 2. 추천 스토리라인

스토리라인은 3개를 고정으로 제안한다.

각 스토리라인은 다음 형식을 쓴다.

- `## N. 제목`
- Quote: 한 줄 요약
- `### 선정 이유`
- `### 슬라이드 구성`
- `### 방송 멘트 초안`

슬라이드 구성에서는 실제 아래 자료 카드 제목과 정확히 맞는 자료명을 코드 텍스트로 인용한다.

예:

`유가 100달러와 이란 리스크` -> `UAE의 OPEC 탈퇴` -> `XLE/CVX/XOM 일봉`

## 3. 자료 수집

자료 수집은 0421 예시를 기준으로 세 덩어리로 고정한다.

### 1. 시장은 지금

고정 순서는 다음과 같다.

1. Finviz 주요 지수 흐름 2장
2. S&P500 히트맵
3. 러셀 2000 히트맵
4. 미국 10년물 국채금리
5. WTI
6. 브렌트
7. DXY
8. 원/달러
9. 비트코인
10. CNN 공포탐욕지수
11. 오늘의 경제 일정

시장지표 Datawrapper 제목은 `지표명: 최신값(전일 대비 변화)` 형식이다.

예:

- `원/달러: 1,488.50원(+16.51원)`
- `미국 10년물 국채금리: 4.418%(+0.064%p)`

부제는 `YY.MM.DD HH:MM 기준`으로 쓴다.

경제 일정은 두 장으로 나눈다.

- `오늘의 미국 경제 일정`: 미국 2★ 이상
- `오늘의 글로벌 경제 일정`: 미국 제외 3★

### 2. 오늘의 이모저모

이모저모 자료 카드는 다음 형식을 쓴다.

- `### 짧은 자료명`
- Quote: 방송에서 왜 볼 만한지 한 문장
- 출처 링크
- 작성 시점
- 이 자료가 여는 질문
- 다음에 붙일 자료
- 원문 핵심 문장 또는 이미지

자료명은 가능한 짧게 쓴다. 스토리라인에서 참조하기 쉽도록 제목은 안정적으로 유지한다.

보강 후보는 메인 자료와 중복되지 않게 하며, 장기적으로는 접기(toggle)나 하단 참고로 내린다.

### 3. 실적/특징주

고정 순서는 다음과 같다.

1. 이번 주 실적 캘린더
2. 실적 캘린더 기반 후보
3. Finviz 일봉/핫뉴스

Finviz 특징주는 티커별로 다음 형식을 쓴다.

- `#### TICKER`
- Quote: 티커를 봐야 하는 당일 화제
- 출처: Finviz
- 최근 뉴스 1줄
- 일봉 이미지

기본 노출 목표는 장기적으로 5개 안팎이다. 0430처럼 실험 목적일 때는 10개까지 허용한다.

## 4. 현재 남은 포맷 리스크

- Notion publisher가 `####`를 완전한 heading block으로 보존하지 못하는 경우가 있다.
- 이미지가 많아지면 페이지가 급격히 길어진다.
- Quote에 원문이 그대로 들어가면 0421 예시보다 무거워 보인다.
- 보강 후보가 메인 자료와 중복되면 진행자용 대시보드 느낌이 약해진다.

## 5. 다음 포맷 개선 방향

- `compact` 모드 추가: 원자료 6개, 특징주 5개, 보강 후보 접기.
- 카드 제목 중복 제거.
- 키워드 표는 감지량보다 방송 소재성 중심으로 재정렬.
- 스토리라인 구성의 자료명은 아래 카드 제목과 정확히 일치시킨다.
