## 아직 어색하거나 미완인 것

### 1. 페이지가 길다

현재 발행본은 block count 236개다. 테스트 대시보드로는 풍부하지만, 실제 진행자가 아침에 훑기에는 다소 길다. 0421 예시의 강점은 "보기에 부담이 적다"는 점이었고, 0430은 정보량은 많지만 압축감이 약하다.

개선 방향:
- 상단에는 `오늘의 핵심 3개`, `키워드 표`, `추천 스토리라인 3개`까지만 둔다.
- 원자료 카드는 6개 안팎으로 제한한다.
- 보강 후보는 기본 접기(toggle) 또는 별도 하단 섹션으로 내린다.

### 2. 보강 후보가 중복된다

`UAE의 OPEC 탈퇴`가 메인 이모저모에도 있고 보강 후보에도 다시 등장한다. 정보 보강 의도는 좋지만, 방송 제작자 입장에서는 "이미 본 자료가 왜 또 나오지?"처럼 느껴질 수 있다.

개선 방향:
- 이미 메인 자료로 들어간 후보는 보강 후보에서 제거한다.
- 같은 이슈의 다른 기사라면 `추가 확인 링크`로 1줄만 붙인다.

### 3. 추천 스토리라인이 아직 설명형이다

스토리라인 1, 2, 3은 서로 다른 꼭지로 잘 나뉘었지만, 아직 "이렇게 풀어갈 수 있습니다"라는 설명 문장에 가깝다. 0421 예시는 자료 이름과 슬라이드 구성의 연결이 더 즉각적이었다.

개선 방향:
- 각 스토리라인은 제목을 더 방송 꼭지처럼 만든다.
- 구성 제안에는 코드 텍스트 자료명을 더 촘촘히 배치한다.
- 예: `유가 100달러와 이란 리스크` -> `UAE의 OPEC 탈퇴` -> `XLE/CVX/XOM 일봉`

### 4. 핵심 키워드 표는 유용하지만 아직 거칠다

감지량 기준이라 `AI 인프라/빅테크`가 크게 잡히고, `단신/화제성` 같은 분류는 의미가 넓다. 또한 예시 소재가 다소 잘려서 사람이 읽는 표로는 아쉽다.

개선 방향:
- 키워드 표는 count보다 `방송 소재성` 점수를 함께 둔다.
- 예시 소재는 짧은 한국어 제목으로 재작성한다.
- 키워드 범주는 매일 고정 6개 정도로 줄인다.

### 5. 특징주 섹션은 여전히 다소 무겁다

Finviz 일봉 10개가 들어가면 페이지 길이가 크게 늘어난다. 특징주는 방송자가 직접 찾을 수 있는 영역이기도 해서, 매일 10개 전부를 노션에 넣는 것이 최선인지는 의문이다.

개선 방향:
- 기본은 5개만 노출한다.
- 나머지는 캡처 파일/JSON에는 남기되 노션에는 `추가 후보` 리스트로 둔다.
- 특징주는 스토리라인과 연결된 티커를 우선한다.
  - 에너지 서사: XLE, CVX, XOM
  - AI 실적 서사: GOOGL, MSFT, META, AMZN
  - 소비/결제 서사: V

### 6. 뉴스 선별은 아직 "많이 나온 것"에 약간 치우친다

UAE/OPEC처럼 속보성 있고 방송 소재성이 큰 뉴스는 잘 잡혔지만, 그 전 단계에서는 뻔한 뉴스가 많이 올라왔었다. 결국 중요한 것은 source coverage보다 선별 로직이다.

개선 방향:
- 후보별로 `novelty`, `visualability`, `narrative_fit`, `market_relevance`, `source_quality` 점수를 둔다.
- "진행자가 왜 이걸 골랐나"를 학습하려면 실제 PPT와 비교한 hit/miss/false positive 장부를 축적해야 한다.

## 0430 PPT/스크립트 도착 후 비교할 질문

1. 우리가 고른 3개 스토리라인 중 실제 방송과 겹친 것은 무엇인가?
2. 실제 방송이 고른 단신/환기 소재는 무엇이었고, 우리 수집망에 있었는가?
3. UAE/OPEC이 실제 메인으로 쓰였는가, 단신으로 쓰였는가, 아니면 빠졌는가?
4. AI/빅테크 실적은 실제로 어떤 각도로 쓰였는가?
5. 특징주는 실제 방송에서 어떤 기준으로 선택됐는가?
6. 우리가 넣은 자료 중 실제 진행자에게 부담만 늘렸을 자료는 무엇인가?
7. 반대로 실제 PPT에 있었는데 Autopark가 놓친 소재는 어떤 source에서 나왔을 가능성이 큰가?
