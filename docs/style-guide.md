# Style Guide

이 문서는 프로젝트가 늘어나도 차트 결과물이 하나의 리서치 시리즈처럼 보이게 만드는 공통 기준이다.

## Core Principle

- 차트는 예쁘기보다 먼저 읽혀야 한다.
- 한 장의 차트가 하나의 메시지만 강하게 말해야 한다.
- 같은 종류의 차트는 최대한 같은 문법을 쓴다.

## Chart Build Rules

### Titles

- 제목은 결론형으로 쓴다.
- 가능하면 수치나 비교 방향을 직접 드러낸다.
- 예시:
  - `위폴 예측은 컨센서스보다 실제값에 가까웠다`
  - `이벤트 참여는 레벨 1이 주도했다`

### Subtitles

- 부제에는 비교 기준, 기간, 표본 범위를 넣는다.
- 너무 긴 설명은 부제보다 주석으로 내린다.

### Annotations

- 강조할 점은 1~2개만 남긴다.
- 수치 해석이 오해될 수 있으면 반드시 표본 크기나 기준선을 같이 적는다.

### Source / Notes

- source 표기는 가능하면 고정 포맷을 유지한다.
- 예시:
  - `Source: Wepoll internal data`
  - `Note: Final valid submissions only`

## Visual Consistency

### Colors

- 기본은 중립 회색 + 핵심 강조색 1개 조합으로 간다.
- 실제값, 기준값, 하이라이트 값에는 항상 같은 의미의 색을 반복 사용한다.
- 프로젝트마다 색을 바꾸기보다, 의미마다 색을 고정하는 쪽을 우선한다.

### Number Formatting

- 퍼센트는 소수점 자리수를 통일한다.
- 금액, 조, 억, %, pp 표기는 프로젝트 내에서 한 번 정하면 끝까지 유지한다.
- 축 라벨과 annotation의 반올림 기준이 다르면 안 된다.

### Axes

- 같은 계열 차트는 가능하면 축 범위를 통일한다.
- 비교가 목적이면 0 기준 포함 여부를 의식적으로 결정한다.
- dot plot이나 bar chart는 정렬 기준을 일관되게 유지한다.

## Recommended Reuse Items

반복 사용이 필요한 요소:

- 제목 문체
- subtitle 문장 구조
- source / note 문구 포맷
- 강조색 의미
- 숫자 반올림 규칙
- category 정렬 규칙

첫 몇 개 차트를 만든 뒤, 실제로 쓴 표현과 스타일을 이 문서에 계속 덧붙여 고정한다.
